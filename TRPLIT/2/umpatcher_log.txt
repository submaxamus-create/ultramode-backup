#   - Supported modes: APPEND, CREATE, FULL, INSERT_AFTER, INSERT_BEFORE, REPLACE
#   - TARGET_LINE is required for: INSERT_AFTER, INSERT_BEFORE, REPLACE
#   - Simulation mode now reports what patch type would be applied.
#   - Existing file content is read before insert / replace operations.
#   - Backup is created before target mutation when the target exists.

# Description:
#   - The top grid layout assumed exactly 4 equal columns and used hard-coded
#     widths like `col_w * 2 + cell_gap`.
#   - Any layout change or new resolution could break this assumption.
#   - Future AI should prefer a general grid layout helper that accepts
#     (row, col, col_span, row_span) instead of hard-coding spans.
# Affected Areas: Top grid cell layout where `col_w` and `cell_gap` are used.
#
# AI_PROBLEM_ID: BOTTOM_BUBBLES_FOOTER_SCALING
# Description:
#   - Footer bubbles used: footer_ctrl_width = max(400, min(470, screen.get_width() // 3))
#   - This hard-coded clamping could misbehave on unusual resolutions.
#   - Future AI should consider proportion-based width and/or adaptive breakpoints.
# Affected Areas: Footer layout code, especially `footer_ctrl_width`.

ACTION TYPES (SAFE FIRST)
1. ARCHIVE_CANDIDATE
   - old backup copy
   - duplicate with confirmed canonical sibling
   - stale generated report

2. REVIEW_REQUIRED
   - unknown identity
   - missing header
   - possible lost intelligence
   - sidecar mismatch

3. PROTECT
   - holy grail / mandatory / live canonical file
   - current live module
   - user-designated protected target

4. HARVEST_FIRST
   - old version likely contains unique ideas or wording
   - thinktank-rich donor file
   - sidecar with unmatched intelligence

APPROVAL MODEL
- GREEN = safe archive proposal
- ORANGE = review / harvest first
- RED = do not touch
- BLUE = informational / untested

CONTROL CENTER CONNECTION
- Control Center should read HAMMER proposals.
- User selects one item or a batch.
- System explains:
  - why item is in proposal list
  - what it is related to
  - what would happen next
- Then user approves or rejects.

MINIMUM SAFE DECISION RULES
- If file is THINKTANK or SIDECAR with useful content markers: do not auto-archive.
- If duplicate exists but one copy may contain lost ideas: mark HARVEST_FIRST.
- If file is in mandatory/live path: PROTECT.
- If classification confidence is weak: REVIEW_REQUIRED.
- If item came from prior bad backup root clutter and has clear duplicate/canonical match: ARCHIVE_CANDIDATE.

WHAT HAMMER MUST NOT DO YET
